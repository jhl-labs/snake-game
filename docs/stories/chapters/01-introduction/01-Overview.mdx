## 1.1 AI Agent란?

AI Agent는 **사용자의 의도를 이해하고 자율적으로 작업을 수행**하는 AI 시스템입니다.
단순한 챗봇과 달리, 실제로 파일을 읽고 쓰고, 명령어를 실행하며, 결과를 확인하고 수정합니다.

> **AI Agent는 "추론 → 도구 실행 → 관찰"을 반복하며 자율적으로 작업을 완료합니다.**

### 핵심 특징

| 특징 | 설명 | 예시 |
|------|------|------|
| **컨텍스트 이해** | 프로젝트 구조, 코드 스타일을 파악 | "이 프로젝트는 MVC 패턴을 사용하네요" |
| **자율적 실행** | 파일 읽기/쓰기, 명령어 실행 | 파일 검색 → 코드 수정 → 빌드 실행 |
| **반복적 개선** | 결과를 확인하고 수정 | 빌드 에러 발생 → 원인 분석 → 수정 |

### 기존 도구와의 차이

**GitHub Copilot (자동완성)**
- 현재 커서 위치에서 다음 코드 예측
- 수동으로 탭 키를 눌러 수락
- 컨텍스트: 현재 파일 위주

**AI Agent (Cursor, Claude Code)**
- 자연어로 복잡한 작업 요청
- 자동으로 여러 파일 탐색/수정
- 컨텍스트: 전체 프로젝트, 규칙 파일, 대화 히스토리

### 언제 어떤 도구를 사용할까?

**Cursor가 적합한 경우:**
- GUI 환경을 선호할 때
- 코드 수정과 동시에 미리보기가 필요할 때
- VS Code 익스텐션을 함께 사용할 때

**Claude Code가 적합한 경우:**
- 터미널 환경에서 작업할 때
- 대규모 리팩토링이나 분석 작업
- CI/CD 파이프라인에 통합할 때
