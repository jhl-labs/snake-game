# AI Agent Cheatsheet

> Cursor와 Claude Code 빠른 참조 가이드

## Cursor 단축키

### 기본 AI 기능
| 단축키 | 기능 |
|--------|------|
| `Cmd/Ctrl + K` | 인라인 AI 편집 |
| `Cmd/Ctrl + L` | AI 채팅 열기 |
| `Cmd/Ctrl + I` | Composer 열기 |
| `Cmd/Ctrl + Shift + K` | 터미널에서 AI |

### AI 채팅 내
| 단축키 | 기능 |
|--------|------|
| `@파일명` | 파일 컨텍스트 추가 |
| `@폴더명` | 폴더 컨텍스트 추가 |
| `@Codebase` | 전체 코드베이스 검색 |
| `@Docs` | 문서 검색 |
| `@Web` | 웹 검색 |
| `Tab` | 자동완성 수락 |
| `Esc` | 자동완성 거부 |

### 편집
| 단축키 | 기능 |
|--------|------|
| `Cmd/Ctrl + Shift + E` | AI로 선택 영역 편집 |
| `Cmd/Ctrl + Enter` | 채팅에서 코드 적용 |

## Claude Code 명령어

### 기본 명령어
| 명령어 | 설명 |
|--------|------|
| `claude` | Claude Code 시작 |
| `claude "prompt"` | 프롬프트와 함께 시작 |
| `claude --help` | 도움말 |
| `claude --version` | 버전 확인 |

### 슬래시 명령어
| 명령어 | 설명 |
|--------|------|
| `/help` | 도움말 표시 |
| `/clear` | 대화 초기화 |
| `/compact` | 컨텍스트 압축 |
| `/init` | CLAUDE.md 생성 |
| `/review` | 코드 리뷰 |
| `/pr` | PR 생성 |
| `/commit` | 커밋 생성 |

### 옵션
| 옵션 | 설명 |
|------|------|
| `--verbose` | 상세 출력 |
| `--model` | 모델 선택 |
| `--resume` | 이전 대화 재개 |
| `--print` | 출력만 (실행 안함) |

## 자주 사용하는 프롬프트 패턴

### 파일 작업
```
"src/game.c 파일을 읽어줘"
"include/ 폴더의 모든 헤더 파일을 보여줘"
"snake.c에서 move 관련 함수를 찾아줘"
```

### 코드 분석
```
"이 함수가 하는 일을 설명해줘"
"이 코드의 버그를 찾아줘"
"메모리 누수가 있는지 확인해줘"
```

### 코드 생성
```
"~하는 함수를 만들어줘"
"~에 대한 테스트를 작성해줘"
"~모듈을 추가해줘"
```

### 수정
```
"이 버그를 수정해줘"
"이 함수를 리팩토링해줘"
"네이밍 규칙에 맞게 수정해줘"
```

## 컨텍스트 추가 방법

### Cursor
```
@파일명          - 특정 파일
@폴더명/         - 폴더 내 파일들
@Codebase        - 전체 코드베이스
선택 후 Cmd+L    - 선택한 코드
```

### Claude Code
```
"src/game.c를 읽고 ~해줘"
"include/ 폴더를 분석해줘"
"프로젝트 전체에서 ~를 찾아줘"
```

## 프로젝트 설정 파일

### Cursor
```
.cursor/
├── rules/          # 규칙 (.mdc 파일)
└── commands/       # 커맨드 (.md 파일)
```

### Claude Code
```
CLAUDE.md           # 프로젝트 규칙 (루트)
.claude/
├── agents/         # 커스텀 에이전트
├── skills/         # 스킬 정의
├── commands/       # 커스텀 슬래시 커맨드
├── rules/          # 규칙 파일
└── settings.json   # 로컬 설정
```

## 디버깅 팁

### 빌드 에러
```
"빌드 에러가 발생해: [에러 메시지]"
"이 컴파일 에러를 해결해줘"
```

### 런타임 에러
```
"실행 시 크래시가 발생해"
"Segmentation fault가 발생하는 원인을 찾아줘"
```

### 로직 에러
```
"예상대로 동작하지 않아: [증상]"
"이 함수의 결과가 이상해: [예상 vs 실제]"
```

## 유용한 패턴

### 단계별 진행
```
1. "먼저 관련 코드를 분석해줘"
2. "문제점을 찾아줘"
3. "수정 계획을 세워줘"
4. "수정해줘"
5. "테스트해줘"
```

### 범위 제한
```
"src/snake.c 파일만 수정해줘"
"이 함수만 리팩토링해줘"
"다른 파일은 건드리지 마"
```

### 확인 요청
```
"변경 전에 계획을 보여줘"
"수정 내용을 설명해줘"
"이게 맞는지 확인해줘"
```

## 트러블슈팅

### Agent가 잘못된 코드를 생성할 때
1. 더 구체적인 컨텍스트 제공
2. 기존 패턴 참조 요청
3. 단계별로 진행

### Agent가 규칙을 따르지 않을 때
1. Rule 파일 확인
2. 명시적으로 규칙 언급
3. 예시 코드 제공

### 응답이 너무 길거나 짧을 때
```
"간단하게 요약해줘"
"더 자세히 설명해줘"
"코드만 보여줘"
"설명도 포함해줘"
```
