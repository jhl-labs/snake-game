## 1. 코드 분석 프롬프트

```
1. 주요 함수 목록과 역할
2. 의존하는 다른 모듈
3. 개선이 필요한 부분

```

```
- 호출하는 다른 함수들

```

### 아키텍처 분석

## 3. 코드 생성 프롬프트

### 새 함수
```
다음 요구사항으로 함수를 만들어줘:
- 함수명: snake_can_move
- 파라미터: Snake* p_snake, Direction dir
```

```
- 프로젝트 코딩 스타일 준수
```

### 테스트 코드

## Tips: 좋은 프롬프트 작성법

### DO
- 구체적인 파일/함수명 명시
- 기대하는 결과 설명
- 제약 조건 명시 (예: "다른 파일 수정 금지")
- 단계별 진행 요청

### DON'T
- 모호한 요청 ("좋게 만들어줘")
- 너무 많은 것을 한 번에 요청
- 컨텍스트 없이 요청
